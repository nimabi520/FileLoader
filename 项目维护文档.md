# FileLoader 项目开发维护文档

## 1. 项目概述
本项目是一个基于 Java Swing 开发的桌面端文件自动上传工具。
**核心功能**：实时监控指定文件夹，一旦检测到新文件生成，通过 HTTP POST 协议将其自动上传到配置的服务器。
**主要特性**：
- 支持多文件夹同时监控
- 文件生成完成检测（防止上传未写完的文件）
- 自动队列管理与并发上传
- 实时上传进度条显示
- 历史上传记录查询（批次 ID、状态）
- 可配置的服务器地址与超时参数

---

## 2. 代码结构说明

项目采用扁平化结构，主要类及其职责如下：

### 核心类
- **`MonitorUI.java`** (入口类)
  - **职责**：应用程序的主入口 (`main`)，负责构建 Swing 图形界面。
  - **关键组件**：
    - `uploadQueue`: 待上传文件队列（`BlockingQueue`）。
    - `uploadExecutor`: 负责消费队列并执行上传任务的线程池（默认 3 线程）。
    - `monitorPanel`: 显示表格形式的上传历史（时间、文件名、批次 ID、状态）。
    - `logPanel`: 显示系统运行文本日志。
  - **关键方法**：
    - `performUpload(File)`: 执行上传逻辑，更新 UI 表格和进度条。

- **`FileMonitor.java`** (监控服务)
  - **职责**：使用 Java NIO `WatchService` 监听文件系统事件。
  - **逻辑**：
    1. 监听 `ENTRY_CREATE` 事件。
    2. 检测到新文件后，启动后台线程检查文件“稳定性”（通过文件大小变化判断文件是否写入完成）。
    3. 文件就绪后调用 `MonitorUI.uploadFile()` 加入上传队列。

- **`ProgressUploadService.java`** (上传服务)
  - **职责**：封装 HTTP `Multipart/form-data` 上传逻辑。
  - **逻辑**：不依赖第三方库，使用原声 `HttpURLConnection` 实现，支持进度回调。
  - **修改点**：最近增加了 `batchId` 参数支持，用于关联 UI 显示和后端日志。

- **`AppConfig.java`** (配置管理)
  - **职责**：管理全局配置（服务器 URL、超时时间、UserId）。
  - **存储**：配置文件保存在运行目录下的 `fileLoader.properties`。

### 辅助类
- **`UploadResponse.java`**: 简单的 POJO，用于映射服务器返回的 JSON 结果。
- **`ConfigDialog.java`**: 配置界面的 Swing 对话框实现。
- **`ServerDiagnostic.java`**: 独立的诊断工具，用于测试服务器连通性。

## 3. 核心工作流程

### 3.1 监控与上传流程
1. **启动**：用户在界面添加文件夹 -> `FileMonitor.startMonitoring(Path)`.
2. **检测**：`FileMonitor` 捕获文件创建事件 -> 进入“稳定期检查”循环。
3. **入队**：文件写入完成 -> `MonitorUI.uploadFile()` -> 放入 `uploadQueue`.
4. **消费**：`startUploadWorker` 中的线程从队列取出文件 -> `performUpload()`.
5. **上传**：
   - 生成 `batchId`。
   - 在 UI 表格新增一行记录。
   - 调用 `ProgressUploadService.uploadFile()`。
   - 实时更新进度条。
   - 上传完成，更新表格状态（成功/失败）。

## 4. 维护与扩展指南

### 如何添加新的配置项？
1. 在 `fileLoader.properties` 文件中规划新的 Key。
2. 在 `AppConfig.java` 中添加对应的 `getter/setter` 和默认值常量。
3. 在 `ConfigDialog.java` 中在界面上添加对应的输入框，并在保存时调用 `AppConfig.setXxx`。

### 如何修改上传协议？
如果服务器 API 发生变更（例如增加新的 Form 字段）：
1. 打开 `ProgressUploadService.java` 的 `uploadFile` 方法。
2. 在构建 Query 参数或 Multipart Body 部分添加新的字段写入逻辑。
   ```java
   // 示例：添加新的表单字段
   String extraField = "Content-Disposition: form-data; name=\"extra\"\r\n\r\nvalue\r\n";
   writer.append(extraField);
   ```

### 如何修改界面布局？
界面主要由 `MonitorUI.java` 控制：
- **表格列修改**：找到 `createMonitorPanel()` 方法，修改 `columnNames` 数组。
- **布局调整**：主要布局逻辑在 `initializeUI()` 方法中。

## 5. 常见问题排查

- **文件无法上传**：
  - 检查 `fileLoader.properties` 中的 URL 是否正确。
  - 使用界面上的“诊断”按钮运行 `ServerDiagnostic`。
- **文件重复上传**：
  - 检查 `FileMonitor.java` 中的 `checkingFiles` 逻辑，确保去重判断正常工作。
- **界面卡顿**：
  - 确保耗时操作（如 I/O、网络请求）不要在 Swing 的事件分发线程（EDT）中运行，应提交给 `uploadExecutor` 或新线程。
